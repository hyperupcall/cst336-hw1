Floating Point Instructions

When writing C in a hosted environment (not freestanding), programmers rarely need to think about whether certain CPU instructions should be restricted or forbidden entirely. In kernels like Linux, programmers may need to think carefully, weighing performance, developer experience, and other considerations.

According to the Linux Documentation

Kernel code is normally prohibited from using floating-point (FP) registers or instructions, including the C float and double data types. This rule reduces system call overhead, because the kernel does not need to save and restore the userspace floating-point register state.

Of course, this makes sense. Let's say that a program context includes instructions that call a syscall: execution will move from a more priviledged protection ring level 3 to level 0. Kernel code will execute, storing and retrieving data using the registers made available by the ISA. When execution returns back to userspace, the registers must be “reset” to their original values. This is fundamental as a programming context must continue to work as if exeuction did not pause. Even if the instructions of a program do not use floating point digits, Linux still must “reset” them to some default value. Otherwise, the kernel will leak information to userspace, which is a security issue.

kexec

Typically to boot into an operating system, a user must &ldquot;restart&rdquote; their computer. Linux allows a user to boot from one Linux kernel into another Linux kernel using functionality called kexec. This significantly improves the time it takes to boot because hardware initialization (performed by the BIOS) is skipped.

kexec performs two high-level steps:

  • Loads new kernel in memory
  • Reboots into the loaded kernel

Implementations of Data Structures

Like any larger standalone C program, the Linux kernel implements many commonly used data structures.

  • Linked List
  • Associative Array
  • Min Heap
  • Radix Trees and Sparse Arrays
  • Folio Queue
